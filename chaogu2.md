# A股行情辅助分析项目进度记录

更新时间：2026-07-07

## 当前服务与启动方式

- 推荐从项目根目录运行 `启动行情系统.bat`，一键启动后端和前端。
- 后端地址：`http://127.0.0.1:8787/`
- 前端地址：`http://127.0.0.1:5173/`
- 后端健康检查：`http://127.0.0.1:8787/api/health`
- PowerShell 版 `start-dev.ps1` 仍保留；前端单独启动脚本为 `stock-web/run-web-dev.cmd`。

## 已完成的关键事项

- `stock-api`：FastAPI 后端，统一代理免费公开行情源，并使用 SQLite 缓存。
- `stock-web`：Vite + React + TypeScript 前端，提供首页看盘、自选股、市场排行、板块强弱、异动提醒和个股详情。
- `stock-mcp-server`：保留第一版 MCP server，不作为当前前后端主链路。
- 首页全市场行情优先使用项目内置东方财富快照读取器，多节点 fallback：`push2delay.eastmoney.com`、`push2.eastmoney.com`、`28.push2.eastmoney.com`、`82.push2.eastmoney.com`。
- 东方财富全市场快照可返回约 5870 条 A 股记录，包含最新价、涨跌幅、成交额、换手率、PE、PB、总市值、流通市值和 `f62` 主力净流入。
- SQLite 仅作为本地缓存层；缓存内容来自真实免费行情接口，不是 mock。
- 首页已移除北向资金实时净买额指标，避免 2024 年披露口径调整后误显示 0。
- 首页主力资金已复用东方财富快照字段 `f62` 汇总，不再显示“东方财富待通”。
- 北证50当前使用新浪历史指数兜底；没有实时成交额时显示“暂无”，不伪造金额。
- 市场排行已支持涨幅榜、跌幅榜、成交额榜、换手率榜，并按对应字段排序。
- 市场温度已结构化为强势、震荡、分化、弱势，并带数据可信度说明。
- 异动提醒已带 `confirmed / needsConfirm / unavailable` 可信度，不把首页快照初筛当作最终复核。
- 个股详情已支持 MA、MACD、KDJ、RSI、BOLL；技术指标已抽为共享计算工具。
- 个股分析已输出结构化技术快照，并把技术风险作为辅助提示，不输出买卖点、仓位、目标价或收益承诺。
- 新闻面板已拆分“公告线索”和“相关新闻”；公告线索仍基于免费新闻源标题/来源识别，不等同于官方公告专线。
- 第五批工程优化已完成：ECharts 按需引入、详情页懒加载、图表组件懒加载、状态文案乱码修复和文档待办清理。

## 当前数据链路状态

- 首页 `/api/market/overview`：优先东方财富全市场快照；源站异常时使用缓存或兜底链路，并在 `_dataStatus` 标注。
- 个股详情 `/api/stocks/{symbol}`：quote、finance、moneyFlow、news 分别保留可用性、来源、日期和限制说明。
- 分时和 K线：作为个股详情和技术指标计算基础；如果源站异常或缓存过旧，前端需要降级可信度。
- 资金流：全市场主力资金使用东方财富快照 `f62`；个股资金流仍可能受 AkShare/东方财富接口可用性影响，缺失时按不可用处理。
- Mock fallback：仅用于接口失败兜底，前端必须明确标注，不能冒充真实行情。

## 当前待办

- ECharts 图表包已降级为独立懒加载 chunk；Vite 仍提示该 chunk 超过 500KB，但首页主包已降至约 227KB，详情页包约 21KB。
- 个股公告专线仍未接入；当前只是新闻源中的公告线索识别。
- 个股资金流如果长期缺失，后续可参考首页快照直连方式做字段兜底。
- 北证50实时成交额仍缺，继续显示“暂无”，不伪造。
- 可继续补充构建分包策略，例如手动 `manualChunks`，但优先保持简单可维护。

## 合规边界

- 本项目只做行情辅助分析和风险提示，不做真实交易。
- 可以输出“适合关注 / 保持谨慎 / 暂时回避”等观察倾向。
- 禁止输出明确买入、卖出、仓位、目标价或收益承诺。
- 发现 mock、缓存、字段缺失或接口失败时，必须明确标注。
